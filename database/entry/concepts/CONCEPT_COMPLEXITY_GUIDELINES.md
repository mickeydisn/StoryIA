# Complexity Guidelines for Database Entries
## Progressive Revelation Standards

### Difficulty Level Categories

#### **Beginner Level** (Difficulty: Beginner)
- **Target Audience**: General readers, no technical background required
- **Prerequisites**: None or basic general knowledge
- **Content Depth**: Core definition, basic characteristics, simple analogies
- **Accessibility Score**: 8-10/10
- **Technical Detail**: Minimal, focused on concepts over mechanisms
- **Examples**: Memory Fundamentals, AI Fundamentals

#### **Intermediate Level** (Difficulty: Intermediate)
- **Target Audience**: Educated readers, basic technical understanding
- **Prerequisites**: Basic concepts in relevant field
- **Content Depth**: Extended descriptions, key characteristics, practical applications
- **Accessibility Score**: 6-8/10
- **Technical Detail**: Moderate, includes some mechanisms and dependencies
- **Examples**: Data Compression Fundamentals, Quantum Computing Fundamentals

#### **Advanced Level** (Difficulty: Advanced)
- **Target Audience**: Technical readers, domain expertise beneficial
- **Prerequisites**: Intermediate knowledge in field + related concepts
- **Content Depth**: Detailed technical specifications, dependencies, speculative elements
- **Accessibility Score**: 5-7/10
- **Technical Detail**: High, includes calculations, limitations, and complex relationships
- **Examples**: Codec Proustien, OmniSight, ESR

#### **Expert Level** (Difficulty: High/Expert)
- **Target Audience**: Specialists, deep technical knowledge required
- **Prerequisites**: Advanced understanding + multiple related fields
- **Content Depth**: Comprehensive analysis, edge cases, future implications
- **Accessibility Score**: 4-6/10
- **Technical Detail**: Maximum, includes validation, alternative approaches
- **Examples**: Quantum Mesh, complex system interactions

### Category-Specific Standards

#### **Concepts Category**
- **Basic Concepts**: Beginner-Intermediate (Accessibility 7-9/10)
- **Technical Concepts**: Intermediate-Advanced (Accessibility 6-8/10)
- **Speculative Concepts**: Advanced (Accessibility 5-7/10)

#### **Systems Category**
- **Infrastructure Systems**: Intermediate-Advanced (Accessibility 6-8/10)
- **AI Systems**: Advanced (Accessibility 5-7/10)
- **Complex Networks**: Expert (Accessibility 4-6/10)

#### **Characters Category**
- **Major Characters**: Intermediate (Accessibility 7-9/10)
- **Supporting Characters**: Beginner-Intermediate (Accessibility 8-10/10)
- **Technical Specialists**: Intermediate-Advanced (Accessibility 6-8/10)

#### **Locations Category**
- **Major Locations**: Intermediate (Accessibility 7-9/10)
- **Setting Locations**: Beginner-Intermediate (Accessibility 8-10/10)
- **Technical Facilities**: Intermediate-Advanced (Accessibility 6-8/10)

### Progressive Revelation Requirements

#### **Within Individual Entries**
1. **Introduction**: Simple core definition (Beginner level)
2. **Development**: Extended description with characteristics (Intermediate level)
3. **Technical Depth**: Scientific foundation and dependencies (Advanced level)
4. **Integration**: Story function and relationships (Intermediate level)

#### **Across Entry Categories**
1. **Basic Prerequisites**: Introduced first, referenced by complex entries
2. **Technical Building**: Complex concepts build on established foundations
3. **Reader Progression**: Difficulty increases gradually through story
4. **Cross-References**: Enable jumping between related complexity levels

### Complexity Calibration Rules

#### **Accessibility Score Guidelines**
- **9-10/10**: No prerequisites, universally understandable
- **7-8/10**: Basic prerequisites, generally accessible
- **5-6/10**: Technical prerequisites, requires focus
- **3-4/10**: Expert prerequisites, challenging for general readers

#### **Prerequisite Depth Standards**
- **None**: Core concepts, fundamental ideas
- **Basic**: 1-2 related concepts
- **Intermediate**: 3-5 related concepts, basic technical knowledge
- **Advanced**: 5+ concepts, domain expertise

#### **Information Density Standards**
- **Low Density**: 1-2 core ideas, simple explanations
- **Medium Density**: 3-5 related ideas, moderate technical detail
- **High Density**: 6+ interconnected ideas, complex technical analysis
- **Very High Density**: 10+ integrated concepts, expert-level analysis

### Quality Assurance Metrics

#### **Consistency Checks**
- [ ] Difficulty levels match content complexity
- [ ] Accessibility scores reflect actual readability
- [ ] Prerequisites are actually required knowledge
- [ ] Information density appropriate for category

#### **Progressive Standards**
- [ ] Basic concepts introduced before advanced applications
- [ ] Cross-references create logical knowledge paths
- [ ] Reader can follow complexity escalation
- [ ] Similar entries maintain comparable depth

#### **Balance Validation**
- [ ] Entry depth matches narrative importance
- [ ] Technical detail serves story function
- [ ] Reader progression supported by entry sequencing
