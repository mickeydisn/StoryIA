# Concept Database Schema

This schema defines the required sections and structure for concept database entries (CONCEPT_*.md files).

## Required Sections

### Basic Information
- **Concept Name**
- **Category**
- **Alternative Names**

### Definition & Description
- **Core Definition**
- **Extended Description**
- **Key Characteristics**
- **Scope**

### Scientific/Technical Foundation
- **Scientific Basis**
- **Speculative Elements**
- **Dependencies**
- **Enables/Supports**

### Complexity Assessment
- **Difficulty Level**
- **Prerequisites**
- **Learning Curve**

### Narrative Integration
- **Introduction Method**
- **Explanation Techniques**
- **Reader Comprehension Aids**

### Story Function
- **Plot Role**
- **Thematic Connection**
- **Character Development**
- **Conflict Generation**

### Usage Tracking
- **First Appearance**
- **Referenced In**
- **Key Payoff Scenes**
- **Evolution Over Story**

### Accuracy & Validation
- **Scientific Accuracy**
- **Internal Consistency**
- **External Consistency**
- **Potential Issues**

### Reader Experience Impact
- **Wonder/Awe Factor**
- **Accessibility**
- **Memorability**
- **Reusability**

### Cross-References & Relationships
- **Related Concepts**
- **Related Characters**
- **Related Locations**
- **Related Systems**
- **Timeline Integration**

### Quality Assurance
- **Clarity Check**
- **Necessity Check**
- **Balance Check**
- **Integration Check**
